# /godmode:a11y

Accessibility testing and auditing using WCAG 2.1 AA/AAA compliance checks, automated scanners (Axe, Pa11y, Lighthouse), color contrast analysis, keyboard navigation audit, and screen reader compatibility testing. Every finding includes WCAG criterion, code evidence, and concrete remediation.

## Usage

```
/godmode:a11y                              # Full accessibility audit at WCAG 2.1 AA
/godmode:a11y --aaa                        # Audit against stricter AAA level
/godmode:a11y --component DatePicker       # Audit a specific component
/godmode:a11y --page /checkout             # Audit a specific page
/godmode:a11y --contrast-only              # Color contrast analysis only
/godmode:a11y --keyboard-only              # Keyboard navigation audit only
/godmode:a11y --screen-reader              # Screen reader compatibility focus
/godmode:a11y --fix                        # Auto-fix issues after audit
/godmode:a11y --ci                         # CI-friendly output (exit code 1 on fail)
```

## What It Does

1. Defines audit scope (pages, components, interactive flows)
2. Runs automated scanners:
   - **Axe-core** — rule-based accessibility violations
   - **Pa11y** — HTML_CodeSniffer WCAG compliance
   - **Lighthouse** — accessibility score and audit
3. Runs WCAG 2.1 manual checklist:
   - **Perceivable** — text alternatives, contrast, adaptable content
   - **Operable** — keyboard access, navigation, timing
   - **Understandable** — readable, predictable, input assistance
   - **Robust** — compatible with assistive technologies
4. Deep dives into color contrast, keyboard navigation, and screen reader testing
5. Auto-fixes common issues (missing alt, orphaned labels, missing lang attribute)
6. Produces findings with severity, WCAG criterion, and remediation code

## Output
- Accessibility report at `docs/a11y/<target>-a11y-audit.md`
- Auto-fix commit: `"a11y: <target> — fix <N> accessibility issues"`
- Report commit: `"a11y: <target> — <verdict> (score: <N>/100, <N> findings)"`
- Verdict: PASS / CONDITIONAL PASS / FAIL

## Next Step
If FAIL: Fix MUST FIX items, then re-audit with `/godmode:a11y`.
If PASS: `/godmode:visual` for visual testing, or `/godmode:ship` to deploy.

## Examples

```
/godmode:a11y                              # Full audit
/godmode:a11y --component Modal            # Audit modal keyboard traps
/godmode:a11y --contrast-only              # Quick contrast check
/godmode:a11y --fix                        # Audit then auto-fix
```
